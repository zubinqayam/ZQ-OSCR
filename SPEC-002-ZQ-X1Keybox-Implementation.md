### The Most Important Implementation Rule Moving Forward

**Secrets must never cross language boundaries unprotected.**

Meaning:
- Rust handles cryptographic memory
- Python orchestrates APIs only
- UI never receives raw secrets
- agents never receive tokens
- replay logs never contain plaintext

The highest-risk implementation area is: **Rust ↔ Python runtime bridging** because this is where memory leakage, serialization exposure, pointer misuse, IPC interception, token persistence can occur.

The correct architecture is:
**Rust owns secrets. Python requests capabilities.**
(not: Python handles cryptographic state.)
